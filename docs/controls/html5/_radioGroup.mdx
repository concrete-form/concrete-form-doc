> a group of radio buttons<br />
> **Render multiple: **[`<input type="radio" />`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input/radio)

<Props>
  <CommonProps />

  <Prop icon="cf.png" name="options" type="object">
    Options to render*
  </Prop>

  <Prop icon="cf.png" name="orientation" type={<>"vertical"<br />| "horizontal"</>} defaultValue='"vertical"'>
    Orientation of the checkboxes
  </Prop>

  <Prop icon="cf.png" name="labelPosition" type={<>"top"<br />| "bottom"<br />| "left"<br />| "right"</>} defaultValue='"right"'>
    Position of each label
  </Prop>

  <Prop icon="html5.png" name="itemContainerProps" type={<>object<br />| function</>}>
    Forwarded to <strong>every</strong> option container (<code>{` <label /> `}</code> element).<br />
    The container contains the <strong>radio</strong> and the <strong>label</strong>.
    <br /><br /><strong>Function:</strong> If a function is provided, that function will receive option's props (if available) and must return props for the item container.
  </Prop>

  <Prop icon="html5.png" name="itemLabelContainerProps" type={<>object<br />| function</>}>
    If provided, the label will be wrapped with a <code>{` <div /> `}</code> element with the given props.
    <br /><br /><strong>Function:</strong> If a function is provided, that function will receive option's props (if available) and must return props for the label container.
  </Prop>

  <Prop icon="html5.png" name="{...props}" type="object">
    Forwarded to <strong>each</strong> <code>{' <input /> '}</code> element
  </Prop>
</Props>

#### *Options format

:::tip
See [controls options](./controls-options) for full options specs.
:::

- Options props are forwarded to [`<input />`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input/radio) element.
- Options groups are not supported.
