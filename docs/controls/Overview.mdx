> This is the list of controls that are supported by each UI kit. Check on the sidebar for specific doc.

:::tip
All Controls need to be rendered inside a `<Form />` component in order to learn how to interact with the form.
:::

## Controls

| Control | Description |
| --- | --- |
| **`<Autocomplete />`** | a select control that can be filtered by typing |
| **`<DateTime />`** | a date picker, time picker or both |
| **`<FileInput />`** | a file upload control |
| **`<Input />`** | an input of any type (text, number, password...) |
| **`<Select />`** | a select that handle single or multiple values |
| **`<SingleCheckbox />`** | a boolean control (similar to toggle switch) |
| **`<Slider />`** | a slider to select a number within a range |
| **`<Textarea />`** | a text input on multiple lines |
| **`<ToggleSwitch />`** | a boolean control that can be toggled on or off |
| **`<CustomControl />`** | a control that let you manipulate inputs and outputs |

## Controls group

| Control | Description |
| --- | --- |
| **`<Checkbox />`** | a set of checkboxes |
| **`<CheckRadiobox />`** | a set of radio buttons |

## Form Controls

| Control | Description |
| --- | --- |
| **`<SubmitButton />`** | a submit button that display a spinner while submitting |

## Layout

| Control | Description |
| --- | --- |
| **`<LabelledControl />`** | a wrapper that helps you create a 2 columns form |
