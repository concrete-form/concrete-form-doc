# Formik - API

## `<Form />`

> `<Form />` renders a `<form>` HTML element and create a context for the controls. All controls rendered inside this components will be register automatically with Formik.

### Props

<Props>
  <Prop icon="formik.png" name="initialValues" type="object" required>
    Forwarded to <a href="https://formik.org/docs/api/formik#initialvalues-values" target="_blank"><code>{'<Formik />'}</code></a>
  </Prop>

  <Prop icon="formik.png" name="onSubmit" type="function">
    Forwarded to <a href="https://formik.org/docs/api/formik#onsubmit-values-values-formikbag-formikbag--void--promiseany" target="_blank"><code>{'<Formik />'}</code></a>
  </Prop>

  <Prop icon="formik.png" name="formikProps" type="object">
    Forwarded to <a href="https://formik.org/docs/api/formik" target="_blank"><code>{'<Formik {...formikProps} />'}</code></a>
  </Prop>

  <Divider />
  <CommonFormProps />
</Props>

### Examples
```jsx
import Form from '@concrete-form/formik'
import Input from '@concrete-form/html5/Input'
import SubmitButton from '@concrete-form/html5/SubmitButton'

const App = () => {
  const onSubmit = data => { console.log(data) }

  return (
    <Form initialValues={{ firstName: '' }} onSubmit={onSubmit} formikProps={{}}>
      <Input name="firstName" />
      <SubmitButton>submit</Submit>
    </Form>
  )
}
```

### Formik context

The `Formik` Context is automatically exposed so it's possible to use [`useFormikContext()`](https://formik.org/docs/api/useFormikContext) normally inside `<Form />`.

## Custom field properties

:::info `fieldProps` has no effect when using `Formik`
**Radio**, **Checkbox** and **Select** and handled out of the box! for that reason, `fieldProps` is not needed.
:::

## Validation
https://formik.org/docs/guides/validation

:::caution
field level validation are not yet supported by `Concrete Form` controls. Use [Schema Validation](https://formik.org/docs/guides/validation#validationschema).
:::
